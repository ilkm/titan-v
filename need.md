# 需求说明

> 本文档替代早期不成熟版本，作为当前产品需求基线。实现与验收以本文档和对应 PR 说明为准。

## 1. 产品目标

构建一套可规模化的虚拟机自动化管控平台：

- `titan-center` 作为控制端，负责集中管理物理机器与虚拟机集群。
- `titan-host` 作为被控制端，负责接入中控并在宿主机执行虚拟化与自动化能力。
- 宿主虚拟化后端统一基于 **OpenVMM**。

## 2. 角色与职责


| 组件                    | 核心职责                                      |
| --------------------- | ----------------------------------------- |
| **titan-center（控制端）** | 数据持久化、设备注册管理、远程运维、批量群控、策略下发、配置编排          |
| **titan-host（被控端）**   | 注册接入、OpenVMM 虚拟化执行、规则落地、Lua 运行时、虚拟机底层能力暴露 |
| **Guest VM（来宾）**      | 完全由 `titan-host` 控制；虚拟机内部不安装任何程序           |


## 3. titan-center 需求

### 3.1 数据与状态

- 使用 **SQLite** 作为中心端本地数据库。
- 持久化以下核心数据：
  - 物理机器清单与状态
  - 虚拟机清单与状态
  - 机器/虚拟机配置模板
  - 代理策略与映射关系
  - 批量任务执行记录

### 3.2 设备与远程管理

- 管理被控制端物理机器（增删改查、在线状态、能力状态）。
- 支持远程管理单台物理机器。
- 支持批量群控物理机器（批量启停、批量下发规则、批量执行任务）。

### 3.3 虚拟机管理与群控

- 管理每台物理机器上的虚拟机生命周期（创建、启动、停止、销毁、查询）。
- 支持批量群控虚拟机（跨主机与单主机两种模式）。

### 3.4 代理与网络强制策略

- 支持为**每个虚拟机配置独立代理 IP**。
- 支持 **socket 全局代理** 策略。
- 强制要求：虚拟机任意网络请求都必须经过代理链路（不得直连外网）。

### 3.5 配置下发

- 支持批量配置物理机器配置文件。
- 配置下发需具备：
  - 版本标识
  - 回滚能力
  - 结果回执（成功/失败/失败原因）

## 4. titan-host 需求

### 4.1 注册与接入

- `titan-host` 启动后可注册到 `titan-center`。
- 支持被 center 识别、纳管、下发策略、回传执行状态。

### 4.2 虚拟化与环境伪装

- 基于 **OpenVMM** 实现虚拟化能力。
- 按 center 下发规则对虚拟机环境进行伪装，满足 AB 测试场景要求。

### 4.3 Lua 脚本运行时与能力开放

- Guest VM 内部不依赖预装 Agent 或其他常驻程序，能力由 `titan-host` 侧统一提供给 Lua。
- 能够载入并执行 Lua 脚本。
- 向 Lua 暴露虚拟机底层能力（由 host 提供受控 API）：
  - 获取虚拟机内存
  - 操作虚拟机内容
  - 获取虚拟机内应用信息
  - 操作虚拟机内应用
  - 绘制页面（脚本侧 UI/叠加能力）

### 4.4 输入模拟能力

- 提供**完全模拟物理键盘信号**能力。
- 输入路径需满足“接近物理输入”的产品要求，避免简单用户态事件注入模型。

## 5. 控制面与执行原则

- 中心与宿主通信采用统一控制面协议（当前工程为 QUIC + mTLS + 版本化帧协议）。
- 所有批量任务必须具备：
  - 幂等性
  - 超时控制
  - 可观测日志
  - 失败重试与失败回报

## 6. 验收口径（当前版本）

- 本文档描述的是目标能力基线，不代表所有功能已实现。
- 每次交付以“PR 声明范围 + 本文档对应条目”联合验收。
- 未在 PR 声明的能力默认为未交付。
